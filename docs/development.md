# Development Guide

## Prerequisites

- Go `1.23+`
- Node.js `18+`
- PostgreSQL `15+`
- Redis `7+` for the full local stack
- Docker for containerized local development, preview, and execution workflows

## Local setup

```bash
git clone https://github.com/spencerandtheteagues/apex-build-platform.git
cd apex-build-platform
cp .env.example .env
```

Install dependencies:

```bash
cd backend
go mod download

cd ../frontend
npm install
```

## Important environment variables

Required for normal local backend startup:

- `DATABASE_URL`
- `JWT_SECRET`
- `ENVIRONMENT`
- `PORT`

Recommended for realistic local runs:

- `REDIS_URL`
- `SECRETS_MASTER_KEY`
- at least one AI provider credential, or `OLLAMA_BASE_URL` for local-only runs

The backend will validate secrets strictly in production and more loosely in development/test.

## Running the app

### Manual

Backend:

```bash
cd backend
go run ./cmd/main.go
```

Frontend:

```bash
cd frontend
npm run dev
```

### Docker Compose

```bash
docker compose up --build
```

This starts:

- `postgres`
- `redis`
- `api`
- `frontend`

Optional profiles expose Adminer, Redis Commander, and monitoring services.

## Testing

### Backend

```bash
cd backend
go test ./...
go test -race ./...
go vet ./...
go build ./cmd/main.go
```

### Frontend

```bash
cd frontend
npm run typecheck
npm run lint
npm test -- --run
npm run build
```

### End-to-end

```bash
cd tests/e2e
npm ci
npm run generate
npm test
```

The E2E suite is manifest-driven and lives under [`tests/e2e`](../tests/e2e).

## Runtime config for container deploys

The frontend Docker image no longer relies only on build-time Vite variables. Container deployments read runtime settings from `/config.js`, generated by [`frontend/docker-entrypoint.sh`](../frontend/docker-entrypoint.sh), and the app consumes those values at runtime.

Relevant variables:

- `VITE_API_URL`
- `VITE_WS_URL`

This matters for Render and Docker deployments where API endpoints can differ between environments without requiring a rebuild.
